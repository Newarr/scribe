Scribe simplify handoff

Understand the codebase by following the promise: never lose meeting audio.

This file is for the next agent using the simplify skill. It explains Scribe in plain terms, points to the smallest useful reading set, and names the parts where simplification must not remove important risk.

Product macOS menu bar recorder Core rule audio is the durable asset Mode record only Engines ElevenLabs cloud or Cohere local Output one folder per meeting under ~/Scribe/

Bottom line

Scribe records microphone plus system call audio, saves the audio first, then transcribes it into Markdown. The app is designed like insurance: it should only interrupt when a meeting may be missed, setup is broken, recording is active, audio looks bad, or a recording is running too long.

When simplifying anything in this repo, preserve the safety contract before polishing language or structure.

Plain product model

  • The menu bar icon is the trust surface.
  • The user clicks before capture starts.
  • Recording captures both local voice and other people in the call.
  • Audio is written to disk continuously.
  • Transcription can fail without losing audio.
  • Finder is the database. There is no transcript browser.

Do not simplify away

  • No import flow.
  • No mic only fallback.
  • No silent engine fallback.
  • Every session must end with transcript.md.
  • Cloud sends audio plus keyterms only.
  • ElevenLabs key stays in Keychain.
  • Every Scribe window should be hidden from screen sharing.

Architecture in one pass

Area Meaning Main files What to preserve while simplifying
App shell Menu bar, lifecycle, windows, prompt routing, session start and stop. TranscriberApp/Scribe/AppDelegate.swift, RecordingMenu.swift User action starts capture. Active recording suppresses new start prompts.
Capture ScreenCaptureKit writes mic and system streams to disk. CaptureSession.swift, SCKAudioCaptureSource.swift, AudioFileWriter.swift Both streams required. Writes happen during the meeting, not at the end.
Final audio Raw streams become user facing audio.m4a. AudioFinalizer.swift Audio is finalized before transcription and is preserved on failure.
Transcription Selected engine turns audio into transcript text. TranscriptionWorker.swift, ElevenLabsScribeBackend.swift, CohereMLXBackend.swift The session engine is fixed at start. Retry does not switch engines silently.
Recovery Relaunch scans session folders and resumes safe work. SessionSupervisor.swift, OrphanRecoverer.swift, SessionClaim.swift Never race live capture. One sided audio is failed, not transcribed.
Detection Supported meeting apps are observed, probed, then prompt the user. ProcessWatcher.swift, DetectionEngine.swift, StartPromptCoordinator.swift Calendar labels prompts. Calendar alone should not start prompts.
Setup Permission, engine, output folder, and local model readiness checks. PermissionDoctor.swift, SettingsStore.swift, LocalModelManager.swift Missing required setup blocks recording loudly.

Happy path in simple words

  1. User clicks Record or accepts a meeting prompt.
  2. App checks setup and disk space.
  3. App creates a session folder under ~/Scribe/.
  4. Capture writes mic.m4a.partial and system.m4a.partial.
  5. Stop finalizes partial files into mic.m4a and system.m4a.
  6. Worker creates audio.m4a.
  7. Worker transcribes with the selected engine.
  8. Writer saves transcript.md and metadata.json.
  9. Raw streams are deleted after success unless raw retention is on.

Main mental trap

Do not describe transcription as the product. The product is the saved record of the meeting. Transcription is a second step that can fail safely.

Current drift to explain carefully

  • The code now defaults to ~/Scribe/, so old spec drift text about ~/Documents/Scribe/ is stale.
  • The start prompt modal has two buttons, but the modal does not show the full More options disclosure.
  • Quiet Meetings and recurring meeting suppression are missing.
  • The transcript writer still has contract drift around schema, aliases, notes, attendees, and speaker grouping.
  • The active popover does not render real MIC and SYS meters even though live levels are tracked.
  • Recents currently show 3 rows, not 5, and lack inline Open Folder and Open Transcript actions.

Recovery risks to keep visible

  • Append failure cleanup may leave a claim held until exit or staleness.
  • Stop failure can leave the current run without an immediate failed transcript.
  • A crash before the worker can leave a simple pending stub that does not match the final Markdown contract.
  • Retry currently expects audio.m4a, even if raw streams survived and audio.m4a did not.

Execution details

Step Do Done when
1. Set scope Pick one output before editing: a shorter spec section, a cleaned drift list, recovery notes, setup copy, start prompt copy, or transcript contract notes. The target file and section are named, and the work is not mixed with code fixes.
2. Read the source set Read AGENTS.md, docs/spec/SPEC.md, docs/working/CODEBASE-EXAMINATION-REPORT.md, and the nearby code only when a fact needs verification. Every simplified claim can be traced to the spec, the examination report, or current code.
3. Split facts from drift Keep intended product rules separate from current implementation gaps. Do not phrase missing work as shipped behavior. A reader can tell what Scribe promises, what the code does today, and what still needs follow-up.
4. Rewrite around user outcomes Group details by what happens to the recording: starts, blocks, saves audio, transcribes, recovers, or fails with audio preserved. The simplified version follows the meeting audio from user click to final folder.
5. Preserve exact contracts Keep names, storage paths, engine rules, Keychain rules, privacy rules, and transcript requirements exact. No simplification weakens the safety contract or invents a fallback.
6. Verify the result Do one final pass against this page's safety list and drift list before calling the work done. The result is shorter, but still names every risk that can lose audio, hide recording state, or mislead recovery.
